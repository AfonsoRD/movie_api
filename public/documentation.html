This page will interact with the database to show some movies on my REST api


Features

  • Return a list of ALL movies to the user
  • Return data (description, genre, director, image URL, whether itâ€™s featured or not) about a single movie by title to the user
  • Return data about a genre (description) by name/title
  • Return data about a director (bio, birth year, death year) by name
  • Allow new users to register
  • Allow users to update their user info
  • AAllow users to add a movie to their list of favorites (showing only a text that a movie has been added)
  • Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed)
  • Allow existing users to deregister (showing only a text that a user email has been removed)

API Endpoints

Buisness Logic URL HTTP Method Query Parameters Request data format Response data format
Get all movies /movies GET - - A JSON object holding data about all the movies
Get a single movie by title /movies/:title GET :title - A JSON object holding data about a single movie
Get by Genre's Name /movies/genre/:genreName GET :genreName - A JSON object holding data about the genre of a movie
Get by Director's Name /movies/directors/:directorName GET :directorName - A JSON object holding data about the data of a director
Allow users to Register /users POST - A JSON object holding data about new user. Structure: { "name" : "user", "favorite movie" : [] } A JSON object holding data about the user that was added. Structure { "name" : "user", "favorite movie" : [] } and ID auto generated included
Allow users to update info /users/:id PUT :id - A JSON object holding data about the updated information
Add a movie to favorite list /users/:id/:movieTitle POST :movieTitle - A text message indicating whether the Movie was added
Delete movie from favorite list /users/:id/:movieTitle DELETE :movieTitle - A text message indicating whether the Movie was removed
Allow existing users to deregister /users/:id DELETE :movieTitle - A text message indicating whether the User has deregistered
